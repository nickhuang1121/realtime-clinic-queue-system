# 即時診所叫號系統（Realtime Clinic Queue System）

本專案為一個使用 Firebase 建立的即時叫號系統，  
模擬實際診所叫號流程，並整合「病人查詢端」與「診所管理端」於同一畫面，讓使用者能快速理解整體運作流程。

## 📌 專案介紹

本專案以「實際診所叫號情境」為出發點設計，  
並非單純 CRUD 練習，而是朝向「可應用於實際流程的模組化工具」進行開發。

系統主要分為兩個角色：

### 👤 病人查詢端
- 輸入號碼即可查詢目前叫號進度
- 即時顯示目前叫到的號碼
- 根據等待狀態提供提示（如：即將輪到）
- 顯示過號名單與更新時間

### 🏥 診所管理端
- 控制叫號流程（+1、+2）
- 支援過號呼叫
- 可重置資料
- 顯示操作紀錄
- 透過 Firebase Authentication 控制寫入權限

---

## ✨ 功能特色

### 病人端
- 即時查詢叫號狀態
- Firestore `onSnapshot()` 即時同步
- 顯示目前叫號
- 顯示個人號碼與等待狀態
- 過號資訊提示
- 最後更新時間顯示

### 管理端
- 即時叫號控制
- +1 / +2 快速操作
- 過號呼叫
- 操作紀錄顯示
- Firebase 登入 / 登出機制

## 🧠 設計概念

本專案刻意將「病人端」與「管理端」整合在同一畫面中，  
目的是讓觀看者（如面試官）能在短時間內理解：

- 使用者如何查詢
- 管理端如何操作
- 資料如何即時同步

在實際商業應用中，兩者應會拆分為不同頁面或裝置。

---

## 🔄 資料流程說明

### 病人端
- 不需登入
- 僅讀取資料
- 使用 `onSnapshot()` 監聽即時變化

### 管理端
- 可即時讀取資料
- 寫入操作需登入驗證
- 更新資料並同步至 Firestore

---

## 🚀 專案延伸性

本系統核心為「Queue（排隊系統）」設計，  
除了診所叫號外，亦可延伸應用至：

- 餐廳候位系統
- 櫃台服務叫號
- 活動排隊管理

透過調整資料結構與規則，即可應用於不同場景。
